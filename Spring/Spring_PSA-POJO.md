Value Object 개념에서의 Java Bean을 말하는 용어로, 클래스 상속, 인터페이스의 구현, 애노테이션의 사용이 강제되지 않은 순수한 Java Beans (객체)를 말합니다.

<br/>

Java EE 개발을 진행하면서 해당 플렛폼에 종속된 무거운 객체들을 만드는 것에 대한 반발으로 나타난 용어이기도 합니다.

<br/>

Field Value와 Getter, Setter 등 만을 포함한 클래스를 인스턴스화 한 것을 POJO라고 합니다.

<br/>

개발에 POJO를 이용할 경우 특정 인터페이스를 구현하거나 상속 받을 필요가 없어 기존 라이브러리를 지원하기가 용이하고, 객체가 가볍습니다.

<br/>

DTO는 각 계층별로 데이터를 전달하는 Domain Object 입니다. 이 객체의 값을 사용하기 위해선 각 계층에선 ObjectMapper (Jackson) 등을 사용하게 되는데, 해당 메서드 (라이브러리)를 사용할 경우 POJO(Java bean)를 만족하지 못한다면, 기본적으로 직렬화를 할 수 없는 제약이 존재합니다.

<br/>

### **PSA (Portable Service Abstraction)**

환경의 변화와 관계없이 추상화된 계층을 사용하여 편리하게 구현할 수 있는 구조를 말합니다.
이 구조는 POJO 원칙을 철저히 따르는 Spring의 기능으로 Spring에서 동작하는 모든 라이브러리
들은 POJO 원칙을 지키게끔 다양한 기술을 하나의 인터페이스로 추상화하여 일관성 있게 제어할
수 있도록 되어 있습니다. (JDBC, Batch..)
